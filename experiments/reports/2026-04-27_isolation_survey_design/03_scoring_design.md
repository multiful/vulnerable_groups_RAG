## 1. 스코어링 모델 한 눈에

```
total_score = Σ_q ( option_score(q) × weight(q) )

stage = stage_from_cutoffs(total_score)   # 1..5
stage = max(stage, 4) if Q9_score >= 5    # safety override
```

핵심:
- **option_score** = 0~6 범위의 정수. "비고립 응답"이 0, "고립적 응답"이 클수록 큰 값.
- **weight** = 그 문항의 max|h| (Cohen's h). 단 Q9는 안전 가중치 1.5배 곱.
- **컷오프** = 시뮬레이션 분포에서 도출된 4개 임계값 (s1_max … s4_max).
- **safety override** = Q9 점수 5 이상이면 단계가 4 미만으로 떨어지지 않게 보정.

가중치 산정 근거는 §3, 컷오프 산정 근거는 §4, 검증 결과는 별도 문서 `04_validation_report.md`.

## 2. 옵션별 점수 (option_score)

각 응답 옵션에 대해 "비고립 응답 = 0, 가장 고립적 응답 = N" 형태로 단조 증가하는 정수 점수를 부여했다. 점수 단계는 옵션 수에 맞춰 다르며, 모든 점수는 데이터의 응답 비율 차이(Cohen's h) 부호와 일치하도록 설계했다(고립군에서 더 자주 나오는 옵션 = 더 높은 점수).

## 3. 가중치 산정 근거

### 3.1 기본 규칙

  weight(q) = max|h|(q) × safety_multiplier(q)

  safety_multiplier = 1.5 if q == Q9 else 1.0

### 3.2 왜 max|h|를 가중치로 쓰는가

CLAUDE.md의 요구는 "평균적인 사람들 대비 동떨어져 있는 사람들에 위험도 더 높이 매기듯이" 가중치를 산정하는 것이다. Cohen's h는 두 비율의 차이를 arcsin 변환으로 정규화한 효과크기이며, 이는 "한 응답이 한 그룹에서 평균 대비 얼마나 동떨어져 있는가"를 정확히 측정한다.

세 가지 후보 가중치 산정 방식을 검토했다.

1. **균등 가중**(weight=1) — 차원별 변별력 차이를 반영하지 못해 변별력 약한 문항이 강한 문항을 희석시킨다. 기각.
2. **확률비**(p_iso/p_non) — 직관적이지만 비대칭이고 분포 끝(p_non→0)에서 불안정. 기각.
3. **max|h|** — 효과크기 표준 척도, 0~∞ 한정 안 됨, 옵션 분포 전체를 반영. 채택.

### 3.3 가중치 합과 정규화

12문항 가중치 합 = 약 10.34. 각 문항의 이론적 최대 가중 점수 합계 = 약 51.77.

명시적인 0~100 정규화는 수행하지 않는다. 컷오프(s1_max … s4_max)가 raw 가중 점수 위에서 정의되므로 정규화 단계가 추가로 필요하지 않다. 운영 단계에서 사용자에게 점수를 노출하지 않고 단계만 보이게 할 계획이므로, raw 점수 자체의 인간 해석 가능성은 핵심이 아니다.

## 4. 5단계 컷오프 (s1_max … s4_max)

### 4.1 컷오프 값

`scoring_cutoffs.json`에 정의된 후보값:

| 임계값 | 값 | 의미 |
|---|---:|---|
| s1_max | **12.71** | score ≤ s1_max → Stage 1 (안정권) |
| s2_max | **18.21** | s1_max < score ≤ s2_max → Stage 2 |
| s3_max | **29.62** | s2_max < score ≤ s3_max → Stage 3 |
| s4_max | **35.55** | s3_max < score ≤ s4_max → Stage 4 |
| (>s4_max) | — | score > s4_max → Stage 5 |

### 4.2 컷오프 산정 방법

원자료의 분포를 시뮬레이션해 도출했다. 시뮬레이션 절차:

1. 각 문항·각 그룹(고립/비고립)의 응답 옵션 분포(주변 분포)에서 옵션을 독립적으로 sampling (per item, per respondent).
2. sampled 응답에 §2의 점수와 §3의 가중치를 곱해 총점을 계산.
3. 고립군 2,000명, 비고립군 8,000명의 합성 표본을 만든 뒤 그 분포의 백분위 기반으로 컷오프 결정.

배정 원칙:

- s1_max = 비고립군의 50백분위 점수 (≈ 비고립군 절반은 이 값 이하 → Stage 1) → "안정권" 정의
- s2_max = 비고립군의 90백분위 (비고립군 안에서도 약간 위험한 상위 10%까지를 Stage 2로)
- s3_max = 고립군의 50백분위 (고립군 중간값 이하는 Stage 3)
- s4_max = 고립군의 85백분위 (고립군 안에서 위험도가 가장 큰 상위 15%만 Stage 5)

### 4.3 컷오프 정당화

이 배정은 다음 의도를 반영한다.

- **Stage 1**은 PRD §7.1의 "취업 안정권"에 대응한다. 비고립군 절반 이하의 점수만 1단계로 분류되므로, 단계 1은 보수적으로 정의된다.
- **Stage 2**는 비고립군 안에서도 일부 사회참여 약화 신호를 보이는 영역. 자격증 추천에서 활동성 자격증을 우선할 수 있는 영역.
- **Stage 3·4**는 두 그룹의 점수 분포가 겹치는 영역에서 분리된다. 자격증 추천에서 학습 강도를 더 가볍게, 단계형 로드맵을 우선해야 하는 영역.
- **Stage 5**는 고립군 상위 15% — 외부 교류가 거의 단절된 상태. 자격증 추천의 진입 자체가 부담일 수 있어, 추천 전에 회복 자원·지원 사업 안내를 먼저 띄우는 게이트가 필요할 수 있다.

### 4.4 단계 의미 (본 문서 한정 후보 정의)

PRD §7.1·§17은 2~4단계의 세부 의미를 reserved로 둔다. 본 문서는 컷오프 산출 결과에 *후보 라벨*을 붙이지만, 이는 정책적 확정이 아니다.

| stage | 후보 라벨 | 시뮬레이션 결과 (iso/non %) |
|---:|---|---|
| 1 | 취업 안정권 (PRD 고정) | iso 0.05% / non 50.0% |
| 2 | 저위험 / 사회참여 가능 | iso 1.65% / non 40.0% |
| 3 | 경미 위험 / 사회참여 약화 | iso 48.3% / non 10.0% |
| 4 | 고위험 / 외부 교류 단절 가까움 | iso 35.0% / non 0.0% |
| 5 | 최고위험 / 은둔에 가까움 (PRD 고정) | iso 15.0% / non 0.0% |

자세한 검증은 `04_validation_report.md`.

## 5. Safety override

### 5.1 규칙

```
if Q9_option_score >= 5:        # "7일 이상" 또는 "거의 매일"
    stage = max(stage, 4)
```

즉 자살/자해 사고가 7일 이상 지속되었다고 응답한 경우, 다른 점수가 낮더라도 단계가 4 미만으로 내려가지 않게 강제 보정한다.

### 5.2 정당화

- 변별력만 보면 Q9는 max|h|=0.590으로 중상위. 그러나 응답자가 "거의 매일 자살 사고"를 보고했다면 이는 임상 위험 신호로, 통계적 변별력과 별개로 단독으로 단계 상향이 필요하다.
- 본 앱의 추천 결과는 자격증/로드맵이지만, Stage 4 이상으로 분류되면 추천 화면 진입 전에 정신건강 위기 자원(예: 1577-0199 자살예방상담전화 등)을 안내하는 가드 화면을 띄우는 정책을 함께 운영해야 한다. 이 가드 화면 자체는 본 문서 범위 밖(`PROMPT_DESIGN.md` 안전 가드와 함께 운영).

### 5.3 한계

- override는 stage 하향만 막을 뿐 stage 상향은 강제하지 않는다(이미 stage 4-5인 경우 그대로 유지). 즉 Q9만으로 모든 응답자를 stage 5로 보내지 않는다 — 이는 false positive 방지를 위함.

## 6. 구현 의사 코드

```python
def compute_stage(answers: dict[str, int]) -> int:
    """answers: { 'Q1': option_score, 'Q2': option_score, ..., 'Q12': option_score }"""
    weights = {
        'Q1':0.682,'Q2':0.830,'Q3':1.026,'Q4':1.250,'Q5':1.168,'Q6':1.066,
        'Q7':0.755,'Q8':0.697,'Q9':0.885,'Q10':0.471,'Q11':0.574,'Q12':0.937,
    }
    total = sum(answers[q] * weights[q] for q in weights)
    if   total <= 12.71: stage = 1
    elif total <= 18.21: stage = 2
    elif total <= 29.62: stage = 3
    elif total <= 35.55: stage = 4
    else:                stage = 5
    if answers.get('Q9', 0) >= 5:
        stage = max(stage, 4)
    return stage
```

운영 코드에서는 옵션 → 점수 매핑 테이블을 별도 파일(`final_survey.csv` 또는 DB)에 두고 본 함수에 정수 점수를 넘긴다.

## 7. 후속 (튜닝 후보)

본 문서의 점수·가중치·컷오프는 *서울시 2022 데이터 분포 기반 초깃값*이다. 운영 도입 후 다음 케이스에 한해 갱신을 검토한다.

- 실제 사용자 응답 분포가 시뮬레이션과 다르면 컷오프를 실데이터 분위로 재조정.
- 정책 결정으로 stage 2~4 의미가 확정되면 본 문서의 후보 라벨을 그 정의에 맞춰 정렬.
- 자격증 추천 결과의 사용자 만족도/탈락률이 단계별로 비대칭이면 단계 경계를 미세 조정.

각 갱신은 본 폴더가 아니라 새 날짜 폴더(예: `experiments/reports/yyyy-mm-dd_isolation_score_retune/`)에 비교 분석을 추가한다.
